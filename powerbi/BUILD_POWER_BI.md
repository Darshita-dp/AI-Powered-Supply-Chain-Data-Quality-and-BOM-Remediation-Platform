# Building the Power BI Report

## Honest status

**Power BI Desktop was not available/operated in the environment that produced this
repository, so no `.pbix` or PBIP artifact exists and no visual validation has been
performed.** What this package contains is the complete, source-controlled build kit:

| Deliverable | Status |
|---|---|
| Analytics marts (7) in dbt + local warehouse | Implemented and tested (`dbt build --select marts`) |
| Semantic-model specification (tables, relationships, RLS, refresh) | Complete (`semantic-model/model-specification.md`) |
| DAX measure catalog (30+ measures) | Complete (`dax/measures.dax`) — authored, not yet executed in a live model |
| Theme JSON | Complete (`theme/bom-guardian-theme.json`) |
| Page-by-page report spec incl. tooltips + drill-through | Complete (`page-specifications/pages.md`) |
| Local CSV data extracts | Generated by `python scripts/export_powerbi_data.py` |
| Power BI Desktop validation | **Pending — requires manual steps below** |

## Build steps (manual, ~45 minutes)

1. Produce data:
   - Local: `python scripts/run_local_pipeline.py` then `python scripts/export_powerbi_data.py`
     → CSVs in `powerbi/data/` (git-ignored).
   - Snowflake: deploy `warehouse/snowflake/` scripts, load, run dbt with the
     `snowflake` target; connect PBI via the Snowflake connector to
     `BOM_GUARDIAN.MARTS` (+ `CORE` dims) with role `BOMG_ANALYST`,
     warehouse `BOMG_WH_XS`.
2. In Power BI Desktop: **File → Options → Preview features → Power BI Project files**
   (save as PBIP so the report stays source-controllable).
3. Get Data → Text/CSV (or Snowflake) → import all tables listed in the model spec.
4. Model view → create the relationships table from the spec (all single-direction).
5. Apply the theme: View → Themes → Browse → `theme/bom-guardian-theme.json`.
6. Create measures: copy each block from `dax/measures.dax` (display folders per the
   comment headers).
7. Build the 7 pages following `page-specifications/pages.md`.
8. RLS: Modeling → Manage roles → add `Plant Steward` per the model spec.
9. Save as PBIP into `powerbi/project/` and commit (PBIP folders are text; do not
   commit any `.pbix` binary or `cache.abf`).
10. Update `PROJECT_STATUS.md`: flip Power BI validation to "validated in Desktop"
    only after the report actually opens and renders.

## Why the React app stays the demo path

The React Command Center / analytics pages render the same marts through the API, so
the project is fully demonstrable without Power BI. Power BI adds the executive/BI
delivery layer once Desktop is available.
